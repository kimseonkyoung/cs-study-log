# 세션 정리 프롬프트 (재사용용)

매 공부 세션이 끝날 때 아래 프롬프트를 그대로 복사해서 Claude에게 전달한다.
회사(개념·강의시청) 세션과 집(문제풀이) 세션에서 각각 다르게 쓴다.

전체 일정은 `01-java-coding-test/curriculum.md` 참고 (몇 주차에 어떤 섹션인지).

---

## 회사 세션 (강의 시청 + 개념 공부) 끝날 때

```
오늘 회사에서 [섹션 번호/이름, 예: 섹션 6 Stack,Queue] 강의 들은 내용 정리해줘.
- 오늘 본 강의는 체크리스트에서 체크 표시해줘
- 예제 코드 포함해서 개념 정리
- 연습용 실습 문제(빈칸/TODO 형태)도 같이
- cs-study-log/01-java-coding-test/{섹션폴더}/notes.md 의
  "## 개념 정리" 아래에 이어붙이는 형태로 정리해줘
```

## 집 세션 (프로그래머스 문제풀이) 끝날 때

```
오늘 [섹션 이름] 관련 프로그래머스 문제 푼 거 정리해줘.
- 문제별로 solutions/ 폴더에 파일 하나씩
  (예: solutions/lv1-올바른괄호.java)
- 파일 상단 주석에 문제 요약 + 접근 방식 + 시간복잡도
- 막혔던 부분이나 실수했던 포인트는 코드 하단에 주석으로
- 오늘 푼 문제 목록과 배운 점을 해당 섹션 notes.md의
  "## 문제풀이 회고" 아래에 추가해줘
```

## 주차 마무리할 때 (선택)

```
이번 주에 끝낸 섹션들의 notes.md를 훑어보고, 헷갈리거나 약한 개념 있으면
즉석 퀴즈 3문제 정도 물어봐줘.
```

## 복습 체크포인트 (섹션 2개마다 1회 — review-checkpoints.md 참고)

```
지금까지 끝낸 섹션들 기준으로 복습 체크포인트 해줘.
- 개념 퀴즈 3문제 즉석으로 물어봐줘
- 예전에 푼 문제 중 하나 골라서 코드 안 보고 다시 풀어보라고 해줘
- 끝나고 review-checkpoints.md의 해당 체크포인트 항목에
  오늘 결과(재풀이한 문제, 헷갈렸던 개념) 기록해줘
```

## 강의 진도가 회사 2시간을 넘긴 날 (3, 4주차)

```
오늘 [섹션 이름] 강의 중 [N강까지] 봤어. 집에 가서 나머지 이어보고
문제풀이 시작할 건데, 지금까지 본 데까지만 우선 개념 정리해줘.
```

---

## 원칙
- 개념정리(notes.md)와 문제풀이(solutions/)는 항상 분리해서 관리
- notes.md는 강의 체크리스트 + 이론, 코드는 최소한의 예제만
- solutions/는 문제 풀이 코드 중심, 설명은 주석으로
- 일정이 틀어지면 curriculum.md도 같이 업데이트 요청하기
